                  import gc

                  def print_references(obj: Any) -> None:
                    """
                    Pretty print references to the object.
                    """
                    for ref_index, ref in enumerate(gc.get_referrers(obj)):
                        if isinstance(ref, dict) and '__name__' in ref:
                            ref_name = ref['__name__']
                        else:
                            ref_name = str(ref)

                        print(f'Ref {ref_index+1}: {ref_name}')

                  print_references(obj=parent)

                  >>>
                  Ref 1: Child(name='Michael', parent=Person(name='Alex'))
                  Ref 2: __main__
Reference counting is an automation of memory management.

Example of Multi-Processing in black

                  
                    from concurrent.futures import ProcessPoolExecutor

                    def format_file(file_path):
                        # Code to format a single file
                        pass
                    
                    file_paths = ["file1.py", "file2.py", "file3.py"]
                    
                    with ProcessPoolExecutor() as executor:
                        executor.map(format_file, file_paths)
Key Points: Concurrent Processing: black uses the concurrent.futures.ProcessPoolExecutor to format multiple files in parallel, leveraging multiple CPU cores. No asyncio: black does not use asyncio for asynchronous programming as its primary task (code formatting) is CPU-bound rather than I/O-bound. This approach allows black to efficiently format multiple files concurrently, improving performance on large codebases. These are your private notes for this slide. Per-Interpreter GIL (Subinterpreters)

Per-Interpreter GIL (subinterpreters) can significantly improve concurrency and isolation in Python applications but come with added complexity and potential compatibility issues. They are best suited for applications that can benefit from parallel execution and are designed to handle the intricacies of managing multiple interpreters. Free-Threading with GIL Disabled in Python 3.13

Disabling the GIL in Python 3.13 can enable true parallelism and improve performance for CPU-bound tasks, but it also introduces complexity and potential concurrency issues.
